## 📌 Slide 2 — What is AI-Driven Web App & Product Development?

### The Old Way vs. The New Way

| Dimension | Traditional Web Development | AI-Driven Web Development |
|-----------|----------------------------|--------------------------|
| **How features work** | Rule-based logic written by developers | Intelligent responses powered by AI/ML models |
| **Personalization** | One-size-fits-all UI | Every user gets a unique, personalized experience |
| **User Support** | Static FAQ pages or human support team | 24/7 AI-powered chat and instant answers |
| **Content Generation** | Manually written by humans | Auto-generated by AI based on user context |
| **Scalability** | Increases cost with users | AI handles unlimited users at near-zero marginal cost |

### What Makes a Web App "AI-Driven"?

An **AI-driven web app** is a software product that integrates one or more AI/ML capabilities to enhance user experience, automate tasks, or generate intelligent outputs — going far beyond what traditional rule-based code can do.

## 📌 Slide 4 — The Project We Are Building: SkillPath AI

### What is SkillPath AI?

**SkillPath AI** is a full-stack, AI-powered personalized learning platform. It helps students go from "I want to learn [topic]" to "Here is your exact personalized 12-week plan, and I'm here 24/7 to answer your questions."

```

│  📊 Dashboard tracks progress, streaks, and completions             │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### The 5 Core Features of SkillPath AI

#### 1. 🗺️ AI Roadmap Generator
- Student enters: **Goal** (e.g., "Become a React Developer"), **Skill Level** (Beginner/Intermediate), **Weekly Hours** available
- Groq AI generates a **week-by-week, fully structured learning roadmap**
- The roadmap is saved to MongoDB — the student can track which weeks are done

#### 2. 🤖 AI Doubt Assistant (24/7 Chat)
- A real conversational AI chat interface built into the platform
- Uses Groq's LLM to answer programming questions with code examples, explanations, and analogies
- Maintains **conversation history** so follow-up questions are context-aware
- Like having a personal programming tutor available at 3 AM

#### 3. 💡 AI Project Recommendations
- Based on the student's current skill level, the AI recommends **3 project ideas**
- Each recommendation includes: title, description, technologies to use, and why it fits their level
- Projects range from beginner (to-do app) to advanced (full-stack AI applications)

#### 4. 📊 Progress Dashboard
- Visual representation of the student's learning journey
- **Progress bars** per roadmap, **completion percentage**, **learning streak tracking**
- Admin-managed **resources section** with curated tutorials and articles

#### 5. 🔐 Authentication System
- Secure **JWT-based login and registration**
- Passwords hashed with bcrypt
- Protected routes — the dashboard and AI features are only accessible when logged in

## 📌 Slide 5 — The Tech Stack: What You Will Learn

### Why the MERN Stack?

MERN stands for **MongoDB + Express.js + React.js + Node.js**. It is the most widely used full-stack JavaScript combination in the industry, used by companies like Netflix, LinkedIn, Uber, and hundreds of startups.

The key advantage: **one language (JavaScript) across the entire stack** — frontend, backend, and database queries.

```
┌─────────────────────────────────────────────────────────────────────┐
│                        The Tech Stack                               │
│─────────────────────────────────────────────────────────────────────│
│                                                                      │
│  🌐 FRONTEND (What users see)                                        │
│     React.js + Vite  →  Component-based UI                          │
│     React Router     →  Client-side page navigation                 │
│     Axios            →  Making API calls to backend                 │
│                                                                      │
│  ⚙️  BACKEND (The engine/server)                                      │
│     Node.js          →  JavaScript runtime (runs on server)         │
│     Express.js       →  Web framework for routing and APIs          │
│     JWT              →  Token-based authentication                   │
│     bcrypt           →  Password hashing & security                  │
│                                                                      │
│  🗄️  DATABASE (Where data lives)                                      │
│     MongoDB          →  NoSQL document database (JSON-like)          │
│     Mongoose         →  Schema definitions and data validation       │
│                                                                      │
│  🧠 AI INTEGRATION (The intelligence)                                 │
│     Groq AI API      →  Ultra-fast LLM inference                    │
│     Prompt Design    →  Crafting prompts to get structured output   │
│                                                                      │
│  ☁️  DEPLOYMENT (How it goes live)                                    │
│     Vercel           →  Frontend hosting                             │
│     Render           →  Backend + API hosting                        │
│     MongoDB Atlas    →  Cloud database                               │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

### Why Groq AI (not OpenAI)?

| Feature | Groq | OpenAI GPT-4 |
|---------|------|-------------|
| Speed | Extremely fast (LPU hardware) | Standard speed |
| Free tier | Generous free tier | Limited free access |
| API format | OpenAI-compatible | OpenAI standard |
| Best for | Student projects, real-time chat | Production enterprise |

Groq uses **Language Processing Units (LPUs)** instead of GPUs, making it the fastest LLM inference platform available today. For our use case — generating roadmaps and answering doubts — it's perfect.

## 📌 Slide 7 — How AI-Driven Development Actually Works (The Mental Model)

### The 3 Layers of AI in a Web App

```
Layer 1: UI Layer (React)
├── User types a question in the chat box
├── Click "Send" button triggers a function
└── Shows the AI response in a chat bubble

         ↓ Axios HTTP POST request

Layer 2: Backend Layer (Express/Node)
├── Receives the user's question
├── Formats a smart prompt: "You are a coding tutor. Student asks: [question]"
└── Calls the Groq AI API with this prompt

         ↓ Groq API call

Layer 3: AI Layer (Groq LLM)
├── Receives the structured prompt
├── Generates an intelligent, context-aware response
└── Returns the response as JSON

         ↑ Response flows back up through backend → frontend → user
```

### The Magic Formula: Prompt Engineering

The difference between a mediocre AI app and an excellent one is **how you talk to the AI**. This is called **Prompt Engineering**.

**Poor Prompt:**
```
"Give a roadmap for React"
```
> Returns: A generic, unhelpful roadmap that doesn't match the student's situation.

**Engineered Prompt:**
```
You are an expert coding mentor. A student wants to learn React.
Their current level: Beginner (knows HTML, CSS, basic JS)
Their goal: Build a full-stack web app for a job interview
Their available time: 8 hours per week

Generate a structured 10-week learning roadmap. For each week provide:
- Week number and theme
- 2-3 specific topics to study
- 1 hands-on exercise
- 1 checkpoint/milestone

Format the response as JSON.
```
> Returns: A detailed, personalized, structured plan the student can immediately use.

**The lesson:** AI is a powerful engine. You are the driver. The quality of the output depends entirely on the quality of the input (the prompt).

1. **Who is the user?** (A student trying to learn)

3. **What is the simplest solution?** (A generated roadmap from AI)

This thinking is what separates a developer who writes code from a developer who **builds products**.

## 📌 Slide 13 — Key Concepts Glossary (Orientation Edition)

> These are terms you will hear repeatedly. Know their basic definitions before MC1 begins.

| Term | Simple Definition |
|------|------------------|
| **Component** | A reusable, self-contained piece of UI (like a button, card, or navbar) |
| **API** | A way for two systems to communicate. Your frontend talks to your backend through APIs |
| **REST API** | A standardized way to design APIs using HTTP methods (GET, POST, PUT, DELETE) |
| **JSON** | JavaScript Object Notation — the format data travels in across the internet |
| **Database** | Where your app's data is permanently stored (MongoDB in our case) |
| **Frontend** | Everything the user sees in the browser |
| **Backend** | The server-side logic that processes requests and talks to the database |
| **Full-Stack** | Building both the frontend AND the backend |
| **JWT** | JSON Web Token — a secure way to prove a user is logged in |
| **LLM** | Large Language Model — the AI model that powers chat (e.g., Groq's Llama) |
| **Prompt** | The instruction/message you send to an AI model |
| **Deployment** | The process of making your app publicly accessible on the internet |
| **Repository / Repo** | A folder of code tracked by Git (hosted on GitHub) |
| **npm** | Node Package Manager — tool for installing JavaScript libraries |
| **Environment Variable** | A secret value (like an API key) stored outside your code for security |
